# How Claude Code Powered the CrowAgent™ SDLC

> This document tells the story of how Anthropic's Claude Code was used as an
> agentic development partner throughout the full software development lifecycle
> of CrowAgent™ Platform — from requirements to deployment.

---

## What is Claude Code?

Claude Code is an agentic coding assistant from Anthropic that runs directly
in your terminal or VS Code. Unlike a chat-based assistant, Claude Code can:

- Read and navigate your entire codebase autonomously
- Write, edit, and refactor files
- Run tests and interpret results
- Follow multi-step instructions with human-in-the-loop checkpoints
- Maintain context across a full development session

In this project, Claude Code served as the primary development engine across
all three teams in our AI-augmented SDLC model.

---

## Team 1 — Requirements & Architecture (SpecOps)

**How Claude Code was used:**

Claude Code was given the platform vision and asked to produce a structured
requirements specification from scratch. Starting with a plain-language brief:

> "I want to build a sustainability AI platform for university estate managers
> that uses physics-informed thermal models to calculate energy, carbon, and
> financial outcomes of retrofit interventions."

Claude Code generated:
- Functional requirements across 7 capability areas
- Non-functional requirements (performance, security, accuracy)
- Data schemas for assets, scenarios, financial outputs, emission outputs
- Acceptance criteria for each requirement

**Human-in-loop checkpoint:** The spec was reviewed and annotated before any
code was written. Corrections were fed back to Claude Code conversationally.

**Architecture design:** Claude Code then reviewed the requirements and proposed
the modular architecture: `app/` (UI), `core/` (physics + finance engines),
`services/` (external APIs), `config/` (platform settings), `governance/`
(rules and prompts). This separation of concerns was validated by the team
before implementation began.

---

## Team 2 — DevSecOps

**How Claude Code was used:**

With the architecture approved, Claude Code was given the PRE_EXECUTION_PROMPT
as a global guardrail, then asked to implement each module in sequence.

Key interactions included:

**Physics model:** Claude Code implemented the PINN thermal model using the
methodology from Raissi et al. (2019), with constants drawn explicitly from
BEIS 2023 GHG Conversion Factors. When a constant wasn't in the spec, Claude
Code asked rather than assuming — this was critical for financial accuracy.

**Financial engine:** NPV, IRR, payback period, and cost-per-tonne CO₂ were
implemented with Claude Code citing the source for every formula. The human
reviewer validated outputs against known UK HE sector benchmarks before sign-off.

**AI Advisor integration:** Claude Code scaffolded the Gemini 1.5 Flash
integration, including the tool-use pattern for physics calculations, while
ensuring API keys were loaded exclusively from secrets management — never
hardcoded.

**Security:** Claude Code ran bandit and flake8 autonomously, reported findings,
and proposed fixes. No fixes were applied without human review.

---

## Team 3 — QA & Deployment Governance

**How Claude Code was used:**

End-to-end validation was orchestrated by feeding Claude Code the
E2E_VALIDATION_AGENT prompt. It simulated the full user flow, identified
a session state persistence issue in the portfolio manager, and proposed a fix
that was reviewed and approved before deployment.

The GitHub Actions CI pipeline was generated by Claude Code from the
DEVSECOPS_AGENT prompt, then hand-reviewed to ensure the bandit
configuration matched the project's security posture.

## Key Lessons Learned

| Lesson | Detail |
|--------|--------|
| Context-first always wins | Loading CONTEXT.md before every session saved hours of correction |
| Guardrails prevent drift | PRE_EXECUTION_PROMPT stopped Claude from hallucinating dependencies |
| Human checkpoints are not optional | Two calculation errors were caught at checkpoints 2 and 4 |
| Prompts are reusable assets | The prompt files in `/prompts/` can rebuild this platform from zero |
| BCP is a byproduct of good governance | The discipline of structured prompts means full rebuild is possible |

## Replication Instructions

Anyone can rebuild or rebrand this platform by following `BCP_REBUILD_GUIDE.md`
and feeding the prompt files in `/prompts/` to Claude Code in sequence.

The prompts are designed to be:
- **Self-contained** — each file provides full context for its task
- **Guardrailed** — PRE_EXECUTION_PROMPT prevents hallucination
- **Branding-agnostic** — the rebranding prompt makes customisation safe
